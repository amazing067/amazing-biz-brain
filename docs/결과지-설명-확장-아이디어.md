# 치매검사 결과지 — 설명 확장 + "수정하면 다시 만들어주는" 시스템 아이디어

## 결과 화면 vs 리포터(보고서) 구분

| 구분 | 설명 |
|------|------|
| **마지막 결과 화면** | 웹에서 고객이 보는 화면. **그대로 유지**한다. |
| **리포터(보고서)** | 그 결과를 **리포터 형식으로 업그레이드**해서 고객에게 전달하는 문서(PDF 등). |

**핵심**: 리포터가 결과 화면이랑 **똑같은 내용**이면, 고객 입장에선 “굳이 결과지를 받아볼 필요가 없다”.  
그래서 **결과 화면에 나온 데이터(점수·단계·비용 등)를 기반으로**, 리포터에는 **부가 설명·추가 내용**을 더해 “업그레이드된 문서”로 만들어야 한다.  
→ **나온 결과지를 가지고 우리가 업그레이드해서 고객 전달용 리포터를 만드는 상황**이다.

---

## 현재 상태
- 결과지(리포터) 문구가 **코드 안에 하드코딩**되어 있음 (`report-pages-34.ts`, `report-pages-1-2.ts`).
- 문구를 바꾸려면 코드 수정 → 배포가 필요함.
- PDF는 같은 JSON으로 API 다시 호출하면 새로 생성됨(이미 가능).

---

## 목표
1. **리포터에만** 부가 설명·추가 내용을 넣어서, 결과 화면보다 **더 자세한** 고객 전달용 문서로 만든다.
2. 그 부가 내용을 수정하면, **같은 데이터로 리포터만 다시 만들면** 새 내용이 반영되게 한다.

---

## 아이디어 1: 콘텐츠 파일 분리 (JSON/마크다운)

**구조**
- 프로젝트에 **콘텐츠 전용 파일**을 둠. 예: `content/report-content.json` 또는 `content/report-content.json` + `content/report-extra.md`.
- 여기에 **모든 결과지 문구**를 넣음:
  - 단계별 한 줄 설명 (좋음/보통/양호/주의/위험)
  - 단계별 추천 멘트 (지금 `recommendMsg`에 해당하는 문장)
  - 점수 구간 해석 문구
  - "인지 기능 프로필" 설명, "예상비용" 설명
  - **추가 섹션**: "이 검사는 무엇을 보나요?", "인지기능이란?", "주의사항", "법적 고지" 등 제목+본문 리스트

**동작**
- PDF/HTML 생성 시 이 파일을 읽어서 사용.
- 문구 수정 = **JSON/마크다운만 수정** → 저장 후 관리자에서 "3·4페이지 PDF 만들기" 또는 "전체 PDF" 다시 실행 → **새 문구로 PDF 생성** (= "다시 만들어주는" 시스템).

**장점**: 비개발자도 파일만 편집하면 되고, Git으로 버전 관리 가능.  
**단점**: JSON/마크다운 편집에 약간의 규칙이 필요함.

---

## 아이디어 2: 관리자 화면에서 "결과지 문구 편집"

**구조**
- `/admin`에 **"결과지 문구 관리"** 섹션 추가.
- 입력 필드 예:
  - 단계별(좋음/보통/양호/주의/위험) **설명** + **추천 멘트**
  - 추가 설명 블록 여러 개 (제목, 본문)
  - 법적 고지, 주의사항 등.
- **저장** 시: 서버의 JSON 파일에 저장하거나, 간단한 DB/저장소에 저장.
- **미리보기**: 선택한 JSON으로 "PDF 미리보기" 생성.
- **재생성**: 기존처럼 "보고서 데이터 JSON 업로드 → PDF 만들기" 시, **현재 저장된 문구**로 PDF 생성.

**동작**
- 문구 수정 → 저장 → 같은 고객 데이터로 "PDF 다시 만들기" 클릭 → 새 문구가 반영된 PDF 다운로드.

**장점**: 코드/파일을 건드리지 않고 관리자만으로 문구 변경 가능.  
**단점**: 편집 UI와 저장 API를 만들어야 함.

---

## 아이디어 3: 하이브리드 (파일 + 관리자)

- **기본 문구**는 `content/report-content.json` 같은 파일로 두고, 개발/버전 관리용.
- **관리자**에서는 "현재 문구 불러오기" → 편집 → "문구 저장" 시 파일 덮어쓰거나 별도 override 저장.
- PDF 생성 시: override가 있으면 그걸 쓰고, 없으면 파일 내용 사용.

이렇게 하면 "수정하면 다시 만들어주는" 건 **두 방식 모두**에서 가능:
- 파일 수정 후 재생성
- 관리자에서 수정 후 재생성

---

## "다시 만들어주는" 동작 정리

- **이미 있는 것**: 같은 보고서 데이터 JSON으로 API(`/api/generate-report-pages-3-4`, `/api/generate-full-report`)를 다시 호출하면 새 PDF가 생성됨.
- **부족한 것**: 그때 쓰는 **문구가 코드에 고정**되어 있어서, 문구만 바꿔서 "다시 만들기"가 안 됨.
- **해결**: 문구를 **코드 밖(JSON/파일 또는 관리자 저장)**으로 빼 두고, PDF 생성 시 그 값을 읽어 쓰면,  
  **문구 수정 → (파일 저장 또는 관리자 저장) → 같은 데이터로 PDF 다시 만들기** = 항상 최신 문구로 결과지가 "다시 만들어짐".

---

## 리포터에만 넣을 "부가 설명·추가 내용" (업그레이드용)

**결과 화면에는 없고, 리포터(고객 전달 PDF)에만 넣는 내용**으로 설계하면, “결과지만 받는 가치”가 생긴다.

| 구분 | 예시 내용 (리포터 전용) |
|------|--------------------------|
| **단계별 상세 해석** | "좋음(80~100점): 일상생활과 사회활동에 필요한 인지 기능이 잘 유지되고 있습니다. 권장: 규칙적인 생활과 사회 참여를 이어가 주세요." 등 5단계별 2~3문장 |
| **인지 영역 설명** | 기억력/집중력/언어 등 **각 축이 무엇을 측정하는지**, 일상에서 어떻게 쓰이는지 1~2문장 |
| **이 검사에 대해** | "본 검사는 선별 목적이며, 확진이 아닙니다. 정밀 검사가 필요하면 의료기관을 방문하세요." |
| **예상비용 안내** | "장기요양등급·인지지원 한도는 제도 변경에 따라 달라질 수 있습니다." / "사적 간병비는 지원 대상이 아닐 수 있습니다." 등 |
| **주의사항/면책** | "결과는 참고용이며, 진단·치료를 대체하지 않습니다." |
| **FAQ** | "점수는 어떻게 나오나요?", "나이대 평균과 다른데요?", "다음에는 언제 검사받으면 좋나요?" 등 2~3개 |
| **추천 다음 단계** | 단계별 "이런 점검/상담을 고려해 보세요" (예: 주의·위험 구간일 때 정밀검사·보험 상담 안내) |

이런 항목을 **리포터 전용 콘텐츠**(예: `content/report-content.json`의 `extraSections`)로 두고, **PDF/리포터 생성 시에만** 삽입하면 된다. 결과 화면 렌더링에는 사용하지 않는다.

---

## 추천 진행 순서

1. **1단계**: `content/report-content.json` 구조를 만들고, 지금 하드코딩된 문구를 옮긴 뒤, `report-pages-34.ts` / `report-pages-1-2.ts`에서 이 파일을 읽어 쓰게 변경.  
   → 문구 수정 후 "PDF 다시 만들기"만 하면 새 결과지 생성되는 **기본 시스템** 완성.
2. **2단계**: JSON에 `extraSections`(추가 설명 블록)를 넣고, 3·4페이지 HTML에 해당 섹션을 렌더링.  
   → 결과지가 **더 자세한 설명**을 담게 됨.
3. **3단계**(선택): 관리자 페이지에 "결과지 문구 편집" UI 추가해, 파일 없이도 문구 수정 가능하게.

원하시면 1단계용 **`report-content.json` 예시 구조**와, `report-pages-34.ts`에서 이걸 읽어서 쓰는 **수정 예시 코드**까지 구체적으로 작성해 드리겠습니다.

## 정리: 결과 화면 → 리포터 업그레이드 흐름 (구현됨)

- **마지막 결과 화면**: 웹에서 그대로 유지.
- **리포터**: 관리자만 쓰는 화면에서 **결과 데이터(JSON)** 를 넣으면, **다이어그램 + 월 예상 금액 + 부가 설명**을 **PNG**로 뽑고, 그 이미지들로 **따로 리포터**를 만드는 형식.

### 실제 사용 흐름

1. 이메일 등으로 받은 **치매검사보고서_이름_날짜_데이터.json** 을 관리자 페이지에서 선택.
2. **다이어그램 PNG** · **월 예상 금액 PNG** · **부가 설명 PNG** 를 각각 뽑아 다운로드.
3. 워드/한글/디자인 툴에서 위 이미지 + 필요한 텍스트를 붙여 **고객 전달용 리포터** 제작.
4. 부가 설명 문구는 **content/report-content.json** 에서 수정. 저장 후 다시 부가 설명 PNG를 뽑으면 새 문구가 반영됨.
