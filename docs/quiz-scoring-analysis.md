# 퀴즈 점수 계산 구조 및 개선 아이디어

## 1. 전체 점수 계산 흐름

### 1.1 2단계 구조

1. **문항별 채점**: 각 문항에서 `earned` (0 ~ 해당 문항 만점) 계산
2. **영역별 합산**: 같은 `category`끼리 `catScores`에 earned 누적, `catMax`에 만점 누적
3. **가중치 총점**:  
   `weightTotal += (catScores[c] / catMax[c]) * CATEGORY_WEIGHTS[c]`  
   → **총점 = Σ (영역별 달성률 × 영역 가중치)**  
   → `Math.round(weightTotal)` 이 최종 0~100 점수

### 1.2 영역별 가중치 (총합 100)

| 영역       | 가중치 | 비고           |
|------------|--------|----------------|
| 기억력     | 25     | 치매 진단 핵심 |
| 지남력     | 15     | 시간/장소 인지 |
| 주의력     | 10     | 집중력         |
| 계산력     | 10     | 금전 관리      |
| 언어유창성 | 10     | 전두엽 기능   |
| 집행기능   | 10     | 문제 해결     |
| 시공간     | 5      | 길 찾기 등     |
| 판단력     | 5      | 상황 대처      |
| 작업기억   | 5      | 단기 정보 유지 |
| 억제능력   | 5      | 충동 조절      |
| 반응속도   | 0      | 보조 지표      |
| 시각탐색   | 0      | 보조 지표      |
| 시각추론   | 0      | 보조 지표      |

가중치 0인 영역도 `catScores/catMax`는 계산되지만, 총점에는 기여하지 않음.  
(보고서/영역별 % 표시에는 사용됨)

## 3. 영역별 “세분화” 정도

- **한 문항만 있는 영역**  
  - 지남력(2), 억제능력(3), 계산력(6), 집행기능(7), 시각추론(14), 판단력(15)  
  - 해당 문항이 **이진(0/만점)** 이면 → 영역 달성률은 **0% 또는 100%**만 가능.

- **두 문항 이상이거나, 한 문항이라도 부분점이 있는 영역**  
  - 기억력(1+10): 10 고정 + 0~20 → 33~100%  
  - 주의력(4+12): 0/10 + (0~10) → 0~100% (연속적 가능)  
  - 작업기억(5): 0, 20, 40, 60, 80, 100% (6단계)  
  - 언어유창성(9), 시공간(11), 반응속도(8), 시각탐색(13): 문항 하나지만 부분점 → 0~100% 구간 가능.

그래서 “점검해볼 부분”에서 **0% 또는 100%만 나오는 영역**은  
- 해당 영역에 **이진 문항 1개만** 있고,  
- 그 문항을 틀리거나(또는 미완) 맞거나 한 경우입니다.

## 4. 검증·테스트 포인트

1. **총점 범위**  
   - 모든 문항 만점 → 100점인지  
   - 모든 문항 0점(또는 미완) → 0점인지  
   - 일부만 풀었을 때 `catMax`가 0인 영역은 `weightTotal`에 기여하지 않는지

2. **영역별 %**  
   - `details[영역]` / `maxDetails[영역]` = 해당 영역 달성률  
   - 세분화된 문항(5,8,9,10,11,12,13)에서 중간 점수를 주면 0, 100이 아닌 %가 나오는지

3. **경계값**  
   - word-fluency: 오답 5개 이상 → 0점  
   - card-match: 시도 6회 → 만점, 12회 이상 → 0점  
   - reaction: 400ms 경계, schulte: 18초 경계

4. **테스트 모드**  
   - `?test=result` 등으로 저장되는 `answers`와 위 로직으로 계산한 `total`이 기대와 일치하는지

## 5. 개선 아이디어 (코드 변경 없이 제안)

### 5.1 “0 또는 100”만 나오는 영역 줄이기

- **지남력(시계)**:  
  - 1차 시도 오답 후 재시도 허용 시, 2차 정답이면 부분점(예: 5점)  
  - 또는 제한 시간 내 정답까지 걸린 시간으로 구간별 부분점 (예: 5초 이내 10, 10초 이내 7, 15초 이내 5)

- **억제능력(Stroop)**:  
  - 1문항이면 그대로 두거나, 문항을 2~3개로 늘려서 “맞은 개수/문항 수”로 부분점

- **주의력(기호 개수)**:  
  - 정답 7개일 때, 5개 맞추면 5점, 6개 7점 등 “맞은 개수 비율”로 부분점 (정답 수를 게임에서 전달해야 함)

- **계산력/집행기능**:  
  - 오답이어도 “선택지와의 거리”로 부분점은 구현 부담이 큼.  
  - 대신 **문항 수를 2개로** 늘리면 0 / 50 / 100%처럼 단계가 생김.

- **시각추론(패턴)**:  
  - 정답/오답만 전달 중이면, “오답 선택 횟수”나 “첫 시도 정답 여부”를 넘기면 시도당 감점으로 부분점 가능.

요약: **이진 문항에 “부분 정보”(시간, 맞은 개수, 시도 수)를 넘기고, earned 공식만 바꾸면** 같은 구조 안에서 세분화 가능.

### 5.2 가중치 0인 영역(반응속도, 시각탐색, 시각추론)

- 총점에는 기여하지 않지만, **보고서/점검해볼 부분**에는 사용됨.
- “보조 지표”로만 쓰려는 의도라면 유지해도 됨.
- “총점에 조금이라도 반영”하려면 가중치를 2~3 정도로 두고, 다른 영역을 1~2씩 줄이는 식으로 재배분 가능 (총합 100 유지).

### 5.3 총점 공식의 단순 대안

- 현재: **가중 평균** (영역별 달성률 × 가중치).
- 대안 1: **단순 평균**  
  - 모든 출제 영역의 달성률을 같은 비중으로 평균.  
  - 기억력 비중이 줄어듦.
- 대안 2: **최저 영역 보정**  
  - 한 영역이 극단적으로 낮을 때 총점을 더 깎는 식의 “최저 영역 가중” (복잡도 증가).
- 대안 3: **현행 유지 + 문항만 세분화**  
  - 구조는 그대로 두고, 위 5.1처럼 이진 문항만 부분점으로 바꾸면, 영역 %와 총점 모두 더 연속적으로 변함.

### 5.4 검증용 테스트 시나리오

1. **전만점**: 모든 문항 정답/완료 → 100점, 모든 영역 100%.
2. **작업기억만 부분**: 5번에서 3개만 맞춤 → 작업기억 60%, 나머지 만점 → 총점은 가중치로 98 전후.
3. **언어유창성 감점**: 9번 오답 3개 → 10 - 6 = 4점 → 언어유창성 40%.
4. **카드 시도**: 11번 8회 시도 → 10 - 3 = 7점 → 시공간 70%.
5. **이진만 틀림**: 2,3,4,6,7,14 번만 틀림 → 지남력·억제능력·주의력(기호)·계산력·집행기능·시각추론 0% → 이 영역들만 0/100으로 나오는지 확인.

## 6. 검증 시나리오 (직접 테스트용)

| 시나리오 | 가정 | 기대 결과 |
|----------|------|-----------|
| 전 만점 | 모든 문항 정답/완료 | 총점 100, 모든 영역 100% |
| 5번 작업기억 3개만 맞춤 | [7,3,8,x,x] 등 3개 일치 | 작업기억 earned=6, 영역 60% |
| 9번 오답 3개 | word-fluency wrong=3 | earned = 10-6 = 4, 언어유창성 40% |
| 11번 8회 시도 | card-match attempts=8 | earned = 10-3 = 7, 시공간 70% |
| 2·3·4·6·7번만 틀림 | 시계·Stroop·기호·계산·집행 오답 | 지남력·억제·주의력·계산·집행 0% → 이 5개 영역만 0% |
| 1번만 보고 10번 전부 틀림 | memory viewed, multi-choice 0개 맞음 | 기억력 10+0/30 ≈ 33% |

테스트 방법: 관리자 또는 `?test=result`로 특정 answers 저장 후 결과 화면에서 총점·영역별 % 확인.

## 7. 요약

- **현재 구조**: 문항별 earned → 영역별 catScores/catMax → (catScores/catMax) × 가중치 합으로 총점.  
- **0/100만 나오는 이유**: 해당 영역에 **이진 문항 1개만** 있어서.  
- **개선 방향**:  
  - 이진 문항에 **시간·맞은 개수·시도 수** 등 정보를 넘기고 **부분점 공식**을 도입하거나,  
  - 해당 영역 **문항 수를 늘리거나**,  
  - **가중치 0** 영역을 총점에 살짝 넣을지 정책만 정하면 됨.  
- **테스트**: 위 §6 시나리오로 총점·영역 %가 기대와 맞는지 확인하면, 변경 후 회귀 검증에 그대로 쓸 수 있음.

- **코드 변경 시 권장 순서**:  
  1) 이진 문항 중 하나(예: symbol-count)에만 부분점 도입 → 영역 % 세분화 확인  
  2) 나머지 이진 문항(시계, Stroop, 계산, 집행, 패턴)에 같은 방식 확장  
  3) 가중치 0 영역을 총점에 넣을지 정책 결정 후 수치 조정
